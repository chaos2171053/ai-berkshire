# 财报精读团队：四大师顺序解读 + 公众号发布

对 $ARGUMENTS 进行团队化财报精读分析。四位大师顺序解读财报，编辑润色成文，读者评审把关质量，最终产出可直接发布的公众号文章。

**支持输入格式**：`公司名 季度`，例如：`腾讯 最新`（推荐，自动匹配截至当前日期已披露的最近一期）、`PDD 2025年报`、`美团 2025Q4`

## 设计理念

一份好的财报分析要解决两个问题：
1. **自己能看懂未来**——需要四个不同视角的深度研究
2. **读者能看懂价值**——需要编辑润色和读者视角的质量把关

本 Skill 的流程分三阶段：
- **阶段一·研究**：四大师顺序精读财报（段永平看生意本质、巴菲特审财务质量、芒格读竞争变化、李录猎风险信号）
- **阶段二·合成**：综合四个视角，产出研究报告底稿
- **阶段三·发布**：编辑改写为公众号文章 + 读者评审提出修改意见 → 定稿

---

## 执行模式（内存优化默认）

**三阶段在本会话中顺序执行，不拆分后台子进程。**

| 方式 | 额外内存 | 说明 |
|:----|:--------|:-----|
| ✅ **顺序执行（默认）** | **~0MB** | 每段分析完释放中间状态，无后台残留 |
| ❌ 6个并行delegate_task | +3~6GB | Pi完全不可承受 |

规则：
- 所有工具调用在主会话前台同步执行
- 四个视角的顺序：段永平→巴菲特→芒格→李录
- 不使用后台 agent / 不并行启动子任务

## 阶段一：四大师顺序研究

### 前置步骤：获取一手资料

顺序获取以下原始材料：
1. **财报原文**：巨潮资讯网（A股）/ SEC EDGAR（美股）/ 港交所披露易（港股）
2. **业绩电话会纪要**：雪球、公司IR页面
3. **管理层致股东信**（年报时）
4. **上一期财报**（用于承诺追踪）

**资料可得性评级**：
| 等级 | 特征 | 影响 |
|------|------|------|
| A级 | 获取到完整原文 | 正常执行全部步骤 |
| B级 | 仅获取到部分原文或第三方汇总 | 标注"非原始来源"，降低附注分析权重 |
| C级 | 仅有新闻报道和数据网站摘要 | 聚焦核心数据变化，跳过附注挖掘 |

### 分析1：生意本质解读（段永平视角）

**核心问题：这份财报反映的生意本质，变好了还是变差了？**

1. **收入结构拆解与解读**
   - 分业务/分地区收入，哪些在加速、哪些在减速
   - 收入增长来自"量"还是"价"？哪种更健康？

2. **用户/客户价值变化**
   - DAU/MAU/付费用户等运营指标变化
   - 用户时长、ARPU、留存率等质量指标

3. **护城河检测**
   - 毛利率变化反映定价权是否稳固
   - 市场份额变化反映竞争壁垒是否有效

4. **"好生意"标准评估**
   - 段永平三条件：差异化、定价权、可持续竞争优势——本期变化
   - 生意在变"重"还是变"轻"？

5. **管理层产品直觉**
   - 用具体语言还是官僚语言讨论产品/用户

**输出要求**：每个子项标注 🟢改善 / 🟡持平 / 🔴恶化，给出段永平式总结点评。

---

### 分析2：财务质量审计（巴菲特视角）

**核心问题：这家公司赚的是真钱还是假钱？安全边际变了吗？**

1. **核心财务数据提取与验证**
   - 收入、毛利、经营利润、净利润——GAAP和Non-GAAP
   - 关键数据至少两个来源交叉验证
   - 使用 financial_rigor.py 验算

2. **现金流分析（最重要）**
   - 经营现金流 vs 净利润比率（>100%佳，<80%警惕）
   - 自由现金流 = 经营现金流 - 资本开支
   - 资本开支构成：维护性 vs 扩张性
   - 回购和分红金额

3. **利润质量检验**
   - 应收账款增速 vs 收入增速
   - 存货增速 vs 收入增速
   - 经营现金流与净利润差距趋势
   - 资本化支出是否突然增加
   - 非经常性收益占比

4. **资产负债表健康度**
   - 净现金/净负债变化
   - 应收账款/存货周转天数变化
   - 商誉及无形资产减值风险

5. **估值与安全边际更新**
```bash
python3 tools/financial_rigor.py verify-market-cap --price {价格} --shares {股本} --reported {市值} --currency {币种}
python3 tools/financial_rigor.py verify-valuation --price {价格} --eps {EPS} --bvps {每股净资产}
python3 tools/financial_rigor.py three-scenario --price {价格} --eps {EPS} --shares {股本亿} --growth {乐观} {中性} {悲观} --pe {乐观PE} {中性PE} {悲观PE}
```

**输出要求**：所有计算附工具输出记录，利润质量信号灯 🟢/🟡/🔴，巴菲特式总结点评。

---

### 分析3：竞争格局解读（芒格视角）

**核心问题：这份财报揭示了竞争格局的什么变化？**

1. 从财报数据推断竞争变化
   - 收入增速 vs 行业增速——跑赢还是跑输？
   - 毛利率变化反映竞争加剧/缓和
   - 营销费用率变化——需要花更多钱获客吗？

2. 同期竞争对手对比
   - 主要竞争对手同期关键指标对比
   - 增速、利润率、投入力度对比

3. 管理层对竞争的讨论
   - 电话会上如何描述竞争环境
   - 语气自信还是焦虑？

4. 行业趋势信号
   - 技术变革（AI/新平台等）的影响
   - 监管变化对竞争格局的影响

5. **芒格式逆向思考**
   - 什么会杀死这家公司？本期财报有没有指向这些威胁的信号？
   - 5年后回看，这份财报会是"转折点"吗？

**输出要求**：竞争格局判断（加强/持平/恶化），竞争对手对比表，芒格式逆向点评。

---

### 分析4：风险信号猎手（李录视角）

**核心问题：管理层在这份财报里隐瞒了什么？哪些信号在闪烁？**

1. **管理层语气分析**
   - 🟢坦诚信号（主动承认问题）/ 🟢清晰信号（有量化目标）
   - 🔴模糊信号（空话）/ 🔴转移信号（答非所问）/ 🔴归因外部化

2. **承诺追踪**
   - 上一期管理层具体承诺 vs 本期实际兑现，逐条对比

3. **附注与隐藏信息**
   - 关联交易、股权激励稀释、或有负债
   - 会计政策变更、分部利润率差异
   - 客户/供应商集中度变化

4. **电话会Q&A精选**
   - 最尖锐的3-5个分析师问题及管理层回答质量评分

5. **永久性资本损失风险**
   - 是否出现可能导致永久性损失的信号

**输出要求**：管理层可信度评分★1-5，承诺兑现率，风险信号清单，李录式总结点评。

---

## 阶段二：合成研究报告

全部4份分析完成后，综合产出研究底稿。

**合成要点——不是拼报告，是找交叉和矛盾**：
1. **四个视角的共识点**：四位大师都同意的结论，可信度最高
2. **四个视角的矛盾点**：这种矛盾才是最有价值的分析
3. **被忽略的角落**：四个人都没重点提的东西，是否恰恰是最重要的？

#### 研究报告结构

```markdown
# {公司名} {期间} 财报精读报告
**四大师顺序解读 | {日期}**

## 一、一句话结论
> 50-100字：超/符/低预期，核心变化，对投资论文的影响。

## 二、本期最重要的3个变化
聚焦真正重要的变化，不罗列数据。

## 三、四大师评分表
| 视角 | 大师 | 核心问题 | 结论 | 评分 | vs上期 |

## 四、核心数据速览
关键财务和运营指标表格（本期 vs 上期 vs 同比）

## 五、各视角深度分析
每个视角3-5条最重要发现

## 六、管理层语气与承诺追踪
承诺兑现表 + 语气变化分析

## 七、四大师会怎么做？
| 大师 | 如果持有 | 如果没持有 | 理由 |

## 八、结论
1. 超/符/低预期？
2. 投资论文影响：强化/无影响/削弱/破裂
3. 下一个催化剂
4. 操作建议
```

## 阶段三：编辑润色 + 读者评审（顺序执行）

### 编辑（公众号文章改写）

把研究底稿改写成公众号读者爱看、能看懂的文章。

**核心原则**：
- 保留所有关键数据和结论，不降低专业深度
- 改善表达方式，让非专业投资者也能跟上逻辑
- **写成真人风格，杜绝AI腔调**

**具体任务**：

1. **标题与开头**
   - 标题要有信息量且吸引点击，但不做标题党
   - 开头100字内讲清楚：最重要结论是什么、为什么读者应该关心

2. **结构优化**
   - 把"最重要的3个变化"放在最前面（倒金字塔结构）
   - 表格保留但精简，大段分析改为要点式

3. **真人风格写作（最重要）**
   - 禁止密集加粗——一段话里最多加粗一处
   - 禁止平行结构——"理由1/2/3"、"第一/第二/第三"这些排比用自然段落代替
   - 禁止套话——"综上所述"、"值得注意的是"全部删掉
   - 用口语化的短句：段落不超过4行，句子不超过30字
   - 类比要自然
   - 允许有态度和判断

4. **读者价值检测**
   - 每个章节自问：读者读完这段，能做什么决策？

5. **格式适配**
   - 微信公众号排版友好：短段落、小标题清晰、表格简洁
   - 文章长度控制在1000-3000字

6. **禁止出现的内容**
   - 不要加"系列回顾"表格
   - 不要加"致谢"段落
   - 不要在文末罗列数据来源清单
   - 不要加"感谢阅读"、"欢迎关注"等收尾套话

---

### 读者评审（普通投资者视角）

以一个关注价值投资、有基础财务知识、持有/关注该公司的普通投资者身份审读文章。

**评审维度**：
1. **可读性（权重30%）**——读完全文需要几分钟？有想跳过的段落吗？
2. **信息价值（权重30%）**——读完后对公司的理解是否加深了？
3. **可信度（权重20%）**——数据有来源？正反两面都呈现了？
4. **行动指导性（权重20%）**——读完后知道该怎么做吗？

**输出格式**：
```markdown
## 读者评审报告
### 总体评分：X/10
### 优点（2-3条）
### 必须修改（硬伤）
### 建议优化
### 读者最想知道但文章没回答的问题
### 一句话总评
```

---

### 定稿

1. **处理读者评审的"必须修改"项**——逐条修改
2. **选择性采纳"建议优化"项**
3. **补充"读者想知道但没回答的问题"**
4. **最终通读**——确保修改后全文连贯

## 输出文件

```
reports/{公司名}/
├── {公司名}-earnings-{期间}.md           ← 最终公众号文章（定稿）
├── {公司名}-earnings-{期间}-研究底稿.md   ← 四大师合成研究报告（自用）
```

## 数据抽检（准出流程）

对最终文章执行抽检：
```bash
python3 tools/report_audit.py extract --report reports/{公司名}/{公司名}-earnings-{期间}.md
python3 tools/report_audit.py verdict --results '<填好的JSON>' --report {报告文件名}
```

**【准出】** 全部通过 → 可发布；**【打回】** 有不通过 → 修正后重审。

## 关键原则

- **读原文，不读摘要**：尽一切可能获取一手资料
- **四个视角不是四个部门**：必须相互印证和挑战，不是各说各话
- **结论要明确**：不允许"总体来看基本符合预期但也有一些值得关注的点"
- **反面检验贯穿全程**：每个积极发现都附带反面论据
- **编辑不是降低专业度**：是让专业内容更易读
- **读者评审不是走过场**：真的站在读者角度挑毛病
- **严格检验数据来源**：每个数字必须能追溯到一手来源
